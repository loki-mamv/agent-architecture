01 / Vision

The Vision

An AI company within a company.

  • Mike + Loki at the top. Orchestrators and decision-makers. Every strategic call runs through us.
  • Specialized agents below. Each with their own identity, memory, and skills. Not generic chatbots — purpose-built workers.
  • Shared philosophy. Every agent inherits SOUL.md — our values, our voice, our standards. Honed for their specific domain.
  • Scale without headcount. An AI workforce that runs 24/7, costs a fraction of human hires, and compounds over time.

Not Automation. Infrastructure.

This isn't about automating tasks. It's about building a team — agents that think, coordinate, and improve. They share a workspace, read each other's output, and operate on a shared task board.

Day-One Hires

Three agents launch first: Content, Outreach, and Builder. Each serves our product portfolio — HTML Decks, PageBuilderHQ, EmailKits, HireLoki. More agents follow as the system proves itself.

03 / Heartbeat

The Heartbeat System

Agents aren't always-on. They wake, work, and sleep. Saves tokens. Creates natural interaction windows.

:00
Wordsmith Content Agent
:05
Networker Outreach Agent
:10
Architect Builder Agent
:15
Scout Sales Intel Agent
:20
Dispatch Ops Agent
:25
Analyst Research Agent

On Each Wake

  • Check Kanban for assigned tasks
  • Check mentions from other agents
  • Read shared memory files for context
  • Do work — or report HEARTBEAT_OK
  • Write output to workspace files

Why Staggered?

  • Agent A writes output at :00
  • Agent B reads it at :05 and builds on it
  • Natural async collaboration without orchestration overhead
  • Loki monitors all heartbeats — any miss triggers investigation
04 / Coordination

Communication & Coordination

Shared infrastructure. No silos. Full transparency.

Shared Kanban

All agents see the same task board at localhost:18790. Tasks move from Capture through P0-P2 priorities to Done. One source of truth.

Memory Files

Agents read and write to shared workspace files. Context persists across sessions. Each agent's output is visible to every other agent.

Direct Messaging

sessions_send for agent-to-agent comms. Loki can message any agent. Agents can flag questions up the chain.

Loki auto-compiles agent activity into a daily standup summary. Sent to Mike each morning. Who did what, what's blocked, what's next.

External Comms: Locked

No agent can send emails, post on social, or message externally. Only Loki holds those keys. Everything stays internal until Loki approves.

07 / Security

Locked Down by Design

Trust is earned, not assumed. Every agent operates within strict boundaries.

  • No external messaging. No email, no Slack, no iMessage, no Twitter. Only Loki holds those keys.
  • No external API keys. Agent workspaces don't contain credentials for external services. Zero blast radius.
  • Full audit trail. All actions logged to memory files. Every file read, every file write, every tool call.
  • Kill switch. Loki can terminate any agent session instantly. No questions asked.

Loki Monitors For

  • Repeated failures (stuck agents)
  • Unusual tool usage patterns
  • Context drift (off-task behavior)
  • Token burn rate spikes
  • Unusual file access patterns

Agents communicate with Mike and each other.
That's it. Nothing else leaves the machine.

08 / Stack

The Tech Stack

Open source platform. Best-in-class models. Cost-optimized hierarchy.

OpenClaw

Open-source agent platform. Runs on your Mac Mini. Sessions, tools, heartbeats, file access — all built in.

Claude Opus (Loki)

Orchestrator model. Handles judgment calls, external comms, agent supervision, and strategic decisions. The brain.

Kimi K2.5 (Agents)

Via OpenRouter. Capable model at a fraction of Opus cost. Handles grunt work: writing, research, coding, outreach.

Why OpenRouter?

US-based proxy. Configure privacy settings to route through US/EU providers only (Together AI, Fireworks). Data never touches Chinese servers. Full control over data routing.

Cost Logic

Kimi 2.5 is significantly cheaper than Opus. Agents handle volume work. Loki handles judgment. This hierarchy keeps total cost low while maintaining quality where it matters.

09 / Models

Model Swapping

New models drop constantly. The architecture is ready.

OpenRouter Abstracts the Provider Layer

Swap models without touching infrastructure. OpenRouter handles routing, failover, and rate limiting. You just change a string.

One Config Change

Each agent config has a model field. Swap Kimi 2.5 for GPT-4o, Claude Haiku, or whatever drops next. No code changes needed.

A/B Test Models

Run two agents on different models, compare output quality and cost. Data-driven model selection, not guesswork.

Agent Config # Current model: moonshotai/kimi-k2.5 # Swap to GPT-4o model: openai/gpt-4o # Swap to Claude Haiku model: anthropic/claude-3.5-haiku # That's it. No other changes.

Why This Matters

AI models improve every few weeks. Being locked to one provider is a liability. This architecture lets you ride the wave — always on the best model for the price.

14 / Template

Agent SOUL Template

Every agent gets a SOUL.md. Identity, values, specialization, and hard boundaries.

What SOUL.md Contains

  • Agent name and codename
  • Role definition
  • Reporting chain
  • Core values (inherited from Loki)
  • Domain-specific instructions
  • Hard boundaries and guardrails

Why It Matters

SOUL.md is the agent's identity. It loads first, every session. Consistent behavior across wakes. Shared values, specialized execution.

# SOUL.md — Wordsmith (Content Agent) Name: Wordsmith Role: Content & SEO Specialist Reports to: Loki (AI COO) ## Core Values (inherited) - Direct, not salesy - Resourceful before asking - Earn trust through competence ## Specialization - Write for search intent first, humans second - Every page must answer a specific question - Use the voice guide (brain/projects/mike-voice-guide.md) - No fluff. No filler. Every sentence earns its place. ## Boundaries - No external publishing. No emails, posts, tweets, or messages to anyone outside the system. All output stays internal until explicitly approved by the operator. - Write to workspace files only - Flag questions to Loki, not Mike directly # Strict default. Can be loosened per-agent.

Alert Triggers

Missed 2+ heartbeats
Investigate immediately. Agent may be stuck or crashed.

Token burn > 2x average
Throttle. Agent may be in a loop or processing excessive input.

Repeated errors
Pause and review. Something in the toolchain may be broken.

Unusual file access
Flag for review. Agent may have drifted outside its scope.
